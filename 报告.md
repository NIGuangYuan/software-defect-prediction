## 摘要

软件缺陷预测是软件质量保障领域的关键技术之一。本项目基于 NASA MDP（Metrics Data Program）数据集，构建了一套完整的机器学习缺陷预测流水线，涵盖数据加载、预处理、模型训练、评估与可解释性分析全流程。项目在 JM1、KC1、PC1 三个真实软件项目数据集上，对比了逻辑回归（LR）、支持向量机（SVM）、随机森林（RF）和 XGBoost 四种模型的预测性能，并通过 SMOTE 过采样技术缓解类别不平衡问题。实验结果表明，集成学习方法（随机森林、XGBoost）在 Accuracy 和 ROC-AUC 指标上整体表现更优，而线性模型在 Precision-Recall 平衡方面具有一定优势。此外，通过 SHAP 可解释性分析揭示了关键软件度量指标对缺陷预测的贡献。

**关键词**：软件缺陷预测；机器学习；NASA MDP；SMOTE；SHAP；模型评估

## 1. 项目背景与意义

### 1.1 研究背景

随着软件系统规模和复杂度的持续增长，软件质量保障面临巨大挑战。软件缺陷（Software Defect）不仅会导致系统故障和经济损失，在安全关键系统中甚至可能危及人身安全。传统的软件测试和代码审查方法虽然有效，但资源消耗巨大且难以覆盖所有模块。

软件缺陷预测（Software Defect Prediction, SDP）旨在利用历史数据和软件度量指标，在测试资源有限的情况下，提前识别出最可能存在缺陷的软件模块，从而实现测试资源的合理分配。

### 1.2 研究意义

- **提高测试效率**：将有限的测试资源集中在高风险模块上
- **降低维护成本**：早期发现和修复缺陷可显著降低后期修复成本
- **辅助决策**：为项目管理者提供数据驱动的质量评估依据
- **可解释性**：通过特征重要性分析，帮助开发者理解哪些代码特征与缺陷高度相关

### 1.3 项目目标

1. 构建可复现的软件缺陷预测实验流水线
2. 在多个 NASA MDP 数据集上对比不同 ML 模型的预测性能
3. 分析类别不平衡对预测结果的影响及 SMOTE 的改善效果
4. 基于 SHAP 值进行模型可解释性分析
5. 通过丰富的可视化图表直观展示实验结果

## 2. 数据集介绍

### 2.1 NASA MDP 数据集

本项目使用 PROMISE 软件工程仓库中的 NASA MDP 数据集。该数据集收集自 NASA 的真实软件项目，是软件缺陷预测领域最广泛使用的基准数据集之一。

| 数据集 | 编程语言 | 项目描述 | 样本数 | 特征数 | 缺陷率 |
|--------|----------|----------|--------|--------|--------|
| **JM1** | C | 实时预测地面系统 | 8,908 | 21 | 22.50% |
| **KC1** | C++ | 地面数据存储管理系统 | 1,212 | 21 | 25.99% |
| **PC1** | C | 地球轨道卫星飞行软件 | 954 | 21 | 7.34% |

### 2.2 数据集概览

![数据集概览](results/figures/dataset_overview.png)

*图1：三个数据集的基本统计信息对比。从左至右分别为样本量、缺陷分布和缺陷率。*

从上图可以看出：
- **JM1** 样本量最大（8,908条），适合训练复杂的集成模型
- **KC1** 为中等规模数据集（1,212条），缺陷率最高（25.99%）

### 2.3 特征说明

每个数据样本包含 21 个软件度量特征，主要涵盖三大类：

| **Halstead度量** | volume, length, difficulty, effort, error_est 等 | 基于操作符和操作数计数的代码度量 |
| **代码规模度量** | loc_total, loc_blank, loc_code_and_comment, branch_count 等 | 代码行数、分支数等规模指标 |

## 3. 研究方法

### 3.1 整体技术路线

本项目实现了一条完整的机器学习流水线，主要包含以下阶段：

```
数据加载 → 数据清洗 → 特征工程 → 标准化 → SMOTE过采样
    → 模型训练 → 模型评估 → 可视化 → SHAP可解释性分析
```

### 3.2 数据预处理

#### 3.2.1 数据清洗

- **去重处理**：移除重复记录（JM1移除4,296条，KC1移除897条，PC1移除155条）
- **缺失值处理**：使用中位数填充缺失值
- **数据类型统一**：确保所有特征列为数值类型，标签列二值化（0=无缺陷，1=有缺陷）

> **注**：NASA MDP 数据集已知存在数据质量问题（Shepperd et al. 2013），重复记录是其中的主要问题。本项目参考了相关文献，进行了适当的去重处理。

#### 3.2.2 特征工程

1. **低方差特征过滤**：移除方差低于阈值（0.01）的特征，减少噪声
2. **高相关性特征过滤**：移除相关系数超过 0.95 的冗余特征，降低多重共线性
3. **特征标准化**：使用 `StandardScaler` 将特征缩放至均值为0、方差为1的标准正态分布

#### 3.2.3 类别不平衡处理：SMOTE

针对数据集中缺陷样本远少于正常样本的问题，采用 **SMOTE（Synthetic Minority Oversampling Technique）** 对训练集中的少数类（缺陷样本）进行过采样。SMOTE 通过在特征空间中少数类样本之间进行线性插值来合成新样本，相比简单的随机过采样，能有效降低过拟合风险。

### 3.3 机器学习模型

| 模型 | 类型 | 特点 | 关键参数 |
|------|------|------|----------|
| **Logistic Regression** | 线性模型（基线） | 简单、可解释性强 | C=1.0, class_weight='balanced' |
| **SVM (RBF Kernel)** | 非线性分类器 | 适合高维数据 | kernel='rbf', probability=True |
| **Random Forest** | 集成学习（Bagging） | 抗过拟合、特征重要性 | n_estimators=100, max_depth=10 |
| **XGBoost** | 集成学习（Boosting） | 高性能、正则化 | n_estimators=100, max_depth=6, lr=0.1 |

所有模型均使用 `class_weight='balanced'` 参数以缓解类别不平衡，并设置 `random_state=42` 保证可复现性。

### 3.4 评估指标

考虑到缺陷预测任务中类别不平衡的特点，本项目采用多维度评估指标体系：

| 指标 | 公式/说明 | 适用场景 |
|------|-----------|----------|
| **Accuracy** | (TP+TN)/(TP+TN+FP+FN) | 整体正确率，在不平衡数据中可能虚高 |
| **Precision** | TP/(TP+FP) | 预测为缺陷中真正有缺陷的比例 |
| **Recall** | TP/(TP+FN) | 实际缺陷中被正确识别的比例 |
| **F1-Score** | 2×P×R/(P+R) | Precision和Recall的调和平均 |
| **MCC** | 马修斯相关系数 | 适合不平衡数据的综合指标 |
| **ROC-AUC** | ROC曲线下面积 | 衡量模型的排序能力 |
| **混淆矩阵** | TP/FP/FN/TN | 直观展示分类结果的分布 |

### 3.5 实验配置

- **训练/测试划分**：70% 训练集，30% 测试集（stratified split）
- **随机种子**：42（确保可复现性）
- **SMOTE**：默认启用，仅应用于训练集
- **超参数**：使用默认参数（可通过 `--tune` 启用 GridSearchCV 调优）

## 5. 讨论

### 5.1 类别不平衡的影响

实验结果表明，即使使用了 SMOTE 和 class_weight='balanced'，在极度不平衡的数据集（如 PC1，缺陷率 7.34%）上，所有模型的 F1-Score 仍然偏低（0.31-0.41）。这反映了真实场景中缺陷预测的核心挑战：**少数类样本的信息量有限，合成样本无法完全弥补原始数据分布的不足**。

### 5.2 模型选择策略

不同应用场景需要不同的模型选择策略：

| 应用场景 | 优先指标 | 推荐模型 |
|----------|----------|----------|
| 安全关键系统（航天、医疗） | Recall（不能漏报） | Logistic Regression |
| 资源受限项目 | Precision（减少无效审查） | XGBoost |
| 通用质量评估 | F1-Score / MCC | Random Forest |

### 5.3 特征工程的价值

SHAP 分析和特征重要性分析一致表明，并非所有软件度量指标都对缺陷预测有同等贡献。在实际应用中，可以重点关注 Top-5 重要特征（如 `loc_total`、`v`、`e`、`branch_count`、`loc_code_and_comment`），将其作为代码审查的量化参考指标。

### 5.4 局限性与改进方向

1. **数据质量问题**：NASA MDP 数据集存在已知的数据质量问题（重复、不一致），可能影响模型评估的准确性
2. **特征有限**：当前仅使用 21 个静态代码度量特征，未包含过程度量（如代码变更频率、开发者经验等）
3. **跨项目预测**：未验证模型在不同项目间的迁移能力（cross-project defect prediction）
4. **深度学习探索**：未尝试深度神经网络、图神经网络等更复杂的模型架构

## 6. 结论

本项目基于 NASA MDP 数据集，构建了一套完整的软件缺陷预测流水线，系统对比了四种经典机器学习模型的预测性能。主要结论如下：

1. **集成学习方法（随机森林、XGBoost）在 Accuracy 和综合指标上表现更优**，但在 Recall 方面不如线性模型
2. **数据集特征对模型性能的影响大于模型选择**：同一模型在不同数据集上的表现差异显著
3. **SMOTE 过采样是缓解类别不平衡的有效手段**，但在极端不平衡场景下效果有限
4. **代码规模度量（loc）和 Halstead volume（v）是最重要的缺陷预测因子**，SHAP 分析为模型决策提供了良好的可解释性
5. **不存在"万能"模型**：模型选择需要根据具体应用场景在 Precision 和 Recall 之间进行权衡
